# Honoka Collaboration Guide — 分散式知識採集指南

本文件說明如何利用 Telegram 群組實現「一人分享，多人同步存檔」的協作流程。

## 核心構想
建立一個私人的 Telegram 協作群組，成員在群組中丟出的連結，會自動被每位成員電腦上的 **Honoka Bridge** 抓取並存入各自的本機硬碟中，實現去中心化的知識備份。

## 實現步驟：共用群組方案

### 1. 準備各自的 Bot (關鍵)
由於 Telegram 限制一個 Bot Token 同時間只能由一個程式進行接聽 (Polling)，因此**成員不能共用同一個 Bot Token**。
*   每位成員需透過 `@BotFather` 申請屬於自己的 Telegram Bot。
*   每位成員在自己的 Honoka Bridge 設定中填入各自的 Token。

### 2. 建立共用群組
*   建立一個 Telegram Group。
*   將**所有成員的 Bots** 全部拉進這個群組。
*   **重要設定**：確保將所有 Bots 設為群組管理員，或者關閉「隱私模式」(Privacy Mode)，讓 Bots 能夠讀取群組內的所有消息。

### 3. 配置授權 ID
在 Honoka Bridge 的設定中：
*   確保自己的 Telegram ID 已加入 `telegramAllowedUserIds`。
*   如果是群組協作，也可以將群組內的成員 ID 都加入授權名單。

## 運作流程
1.  **分享**：成員 A 在群組丟出一個 Notion 或網頁連結。
2.  **觸發**：群組內所有的 Bots 同時接收到該訊息。
3.  **同步抓取**：
    *   成員 A 的電腦抓取並存入 A 的硬碟。
    *   成員 B 的電腦抓取並存入 B 的硬碟。
4.  **結果**：所有成員都在第一時間獲得了該內容的本地副本。

## 未來計畫：自動化與 Notion 同步
為了實現更高級的「自動分類保存」，未來的開發方向包括：
*   **Notion 推回機制**：定期將本機硬碟中的 `.md` 檔案推回指定的 Notion Database。
*   **自動分類**：利用 AI 根據內容自動標記類別（例如：法律、技術、生活）。
*   **全局搜索**：透過 Notion 的強大搜索功能，檢索所有成員共同採集的本地文檔。
