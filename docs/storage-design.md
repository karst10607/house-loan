# 房貸助手：工業級 Markdown 存儲架構設計 (Storage Design)

本文件詳細說明本專案採用的「葉子束 (Leaf Bundle)」層次化存儲架構。

## 1. 目錄層次結構 (Directory Hierarchy)

為了確保在處理數千份截取文件時仍能保持高效性能與穩定性，我們採用以下路徑規範：

`[根目錄]/[年份]/[月份]/[日期]-[Slug]/`

### 範例：
`Documents/HouseLoan_Clippings/2026/04/14-github-storage-design/`

*   **年份/月份**：避免單一資料夾下存放過多項目，解決操作系統索引緩慢問題。
*   **Slug**：僅使用小寫英數字、連字號 (`-`)。嚴禁使用空格或特殊符號，確保跨平台同步不亂碼。

## 2. 檔案內容組織 (File Organization)

每個存檔資料夾都是一個獨立的單元：

*   **index.md**：主文件名稱固定，方便程式讀取。
*   **assets/**：存放所有該文章引用的圖片、附件。

## 3. YAML Frontmatter 規範

我們在 Markdown 最上方引入 Frontmatter 機制來分離「路徑」與「語義數據」。

```yaml
---
title: "這是一篇帶有 🔥 火熱符號的原始標題"
url: "https://example.com/article"
clipped_at: 2026-04-14 21:00
tags: [房貸, 新聞]
---
```

### 優點：
1.  **容錯性**：即便資料夾名稱（Slug）簡化了標題，我們依然能在檔案內保留最完整的原始標題（包含特殊符號）。
2.  **檢索性**：未來可自動化生成索引表或進行全文搜尋，Frontmatter 提供結構化的數據來源。
3.  **便攜性**：搬移整個資料夾時，內部的相對路徑（如 `![](./assets/img.png)`）依然保持有效。

## 4. 實作準則

*   **Slugify 演算法**：優先保留標題中的英數字，非英數字一律轉換為 `-`。
*   **路徑檢查**：App 在啟動時會根據使用者設定的路徑自動生成對應的日期子目錄。
