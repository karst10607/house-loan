# X (Twitter) 自動影片歸檔：設計方案

本文件記錄了如何實作「在 X 上點擊愛心 (Like/Fav) 後，Honoka 自動下載影片」的技術方案。

## 方案一：RSSHub 訂閱（推薦：最穩、免帳密）

透過 [RSSHub](https://docs.rsshub.app/) 產生你個人「喜歡的內容」RSS 連結，並讓 Honoka Bridge 定時檢查。

- **原理**：RSSHub 會將你的 X Likes 轉成標準 RSS 格式。Bridge 每隔一段時間讀取 XML，發現新連結就下載。
- **優點**：不需在 Bridge 存儲 X 帳號密碼，不怕被 X 封鎖帳號。
- **缺點**：需要一個可用的 RSSHub 實例（可以自建或用公開的）。
- **實作複雜度**：低。

## 方案二：Chrome 插件即時監控（最快：即點即載）

修改現有的 Honoka Lite 插件，當偵測到用戶在 X 頁面點擊 `.css-175oi2r` (Like 按鈕) 時，觸發動作。

- **原理**：插件監聽網頁上的點擊事件，一旦點擊 Liked，就抓取目前 Tweet 的網址發送給 Bridge。
- **優點**：下載速度最快，點擊瞬間就開始。
- **缺點**：你必須在瀏覽器上操作。如果你在手機上點愛心，電腦沒開瀏覽器就不會觸發。
- **實作複雜度**：中。

## 方案三：後端 Cookie 爬蟲（全自動：但有風險）

在 Bridge 裡模擬瀏覽器行為，使用你的 X Cookie 定時抓取 `/likes` 頁面。

- **原理**：利用手動獲取的 Cookie，讓後端有權限訪問你的隱私頁面。
- **優點**：完全自動，無論在哪裡點愛心（手機、平板）都能抓到。
- **缺點**：X 的反爬機制極嚴格，Cookie 可能會過期，且帳號有被標記為異常的風險。
- **實作複雜度**：高。

---

## 結論與建議

**初期建議採用「方案一 (RSSHub)」**。
我們可以在 `settings.json` 加入一個 `rssFeedUrl` 欄位，只要填入，Bridge 就會自動開啟背景掃描任務。
